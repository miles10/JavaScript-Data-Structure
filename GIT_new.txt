GIT 入门：
1.创建版本库
$ mkdir learngit
$ cd learngit
$ pwd
/Users/michael/learngit
（Windows系统避免使用中文目录）
第二步，通过git init命令把这个目录变成Git可以管理的仓库：
$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/
添加文件，到git中
$ git add readme.txt
用git commit告诉GIT所执行的当前步的说明
$ git commit -m "wrote a readme file"
[master (root-commit) cb926e7] wrote a readme file
 1 file changed, 2 insertions(+)
 create mode 100644 readme.txt
 多个文件的提交，提交文件时要用add与commit两步
 $ git add file1.txt
$ git add file2.txt file3.txt
$ git commit -m "add 3 files."
运行git status命令看看结果：（查看当前状态）
$ git status
查看修过的区别  diff=different
$ git diff readme.txt 
添加提交新文件
$ git add readme.txt
查看状态：
$ git status
提交文件时要用add与commit两步
$ git commit -m "add distributed"
查看状态
$ git status
查看历史
$ git log
或者使用下面，可以减少显示内容，作用也是查看历史
$ git log --pretty=oneline
返回到上一个版本，上一个版本HEAD^，上两个版本HEAD^^，上100个版本HEAD~100
$ git reset --hard HEAD^
查看txt里面的内容
$ cat readme.txt
回到指定的一个，数字为之前的$ git log --pretty=oneline产生的数字，数字只用取版本号的前几位即可
$ git reset --hard 3628164
查看每一次命令，结果中有每一次的版本号，可以根据这个实现回到每一次的内容
$ git reflog
git中有暂存区的概念，意味着git add时先将文件放入暂存区，然后git commit将文件放入当前分支
提交后，用git diff HEAD -- readme.txt命令可以查看工作区和版本库里面最新版本的区别：
$ git diff HEAD -- readme.txt 
修改文件后，还未add进入暂存区，还在工作区时可以用以下命令恢复，如果添加到暂存区了，则恢复到添加到暂存区的状态
$ git checkout -- readme.txt
用命令git reset HEAD file可以把暂存区的修改撤销掉（unstage），重新放回工作区：
$ git reset HEAD readme.txt
工作空间中删除文件
$ rm test.txt
git 暂存区删除文件
$ git rm test.txt
告诉分支，该动作
$ git commit -m "remove test.txt"
删除错误，恢复
$ git checkout -- test.txt


2.远程库
将本地的master分支推送到GIThub上
$ git push origin master
小结
小结

要关联一个远程库，使用命令git remote add origin git@server-name:path/repo-name.git；

关联后，使用命令git push -u origin master第一次推送master分支的所有内容；

此后，每次本地提交后，只要有必要，就可以使用命令git push origin master推送最新修改；
