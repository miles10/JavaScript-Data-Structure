GIT 入门：
1.创建版本库
$ mkdir learngit
$ cd learngit
$ pwd
/Users/michael/learngit
（Windows系统避免使用中文目录）
第二步，通过git init命令把这个目录变成Git可以管理的仓库：
$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/
添加文件，到git中
$ git add readme.txt
用git commit告诉GIT所执行的当前步的说明
$ git commit -m "wrote a readme file"
[master (root-commit) cb926e7] wrote a readme file
 1 file changed, 2 insertions(+)
 create mode 100644 readme.txt
 多个文件的提交，提交文件时要用add与commit两步
 $ git add file1.txt
$ git add file2.txt file3.txt
$ git commit -m "add 3 files."
运行git status命令看看结果：（查看当前状态）
$ git status
查看修过的区别  diff=different
$ git diff readme.txt 
添加提交新文件
$ git add readme.txt
查看状态：
$ git status
提交文件时要用add与commit两步
$ git commit -m "add distributed"
查看状态
$ git status
查看历史
$ git log
或者使用下面，可以减少显示内容，作用也是查看历史
$ git log --pretty=oneline
返回到上一个版本，上一个版本HEAD^，上两个版本HEAD^^，上100个版本HEAD~100
$ git reset --hard HEAD^
查看txt里面的内容
$ cat readme.txt
回到指定的一个，数字为之前的$ git log --pretty=oneline产生的数字，数字只用取版本号的前几位即可
$ git reset --hard 3628164
查看每一次命令，结果中有每一次的版本号，可以根据这个实现回到每一次的内容
$ git reflog
git中有暂存区的概念，意味着git add时先将文件放入暂存区，然后git commit将文件放入当前分支
